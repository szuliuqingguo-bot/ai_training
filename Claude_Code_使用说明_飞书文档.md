# Claude Code 使用说明

> 📘 适用人群：Claude Code 新手到进阶用户 | 更新日期：2026-07-03

---

## TL;DR

Claude Code 是 Anthropic 推出的本地 AI 编程助手（简称 CC），基于大模型循环（LM Loop）持续理解上下文、生成代码、执行验证。核心优势是**本地权限**和**优秀的工具调用 Harness**。从写代码到文案整理、数据分析都能胜任。本文档覆盖安装配置、交互技巧、上下文管理、高阶扩展的完整学习路径。

---

## 一、基础认知

### 1.1 Claude Code 是什么

Claude Code 于 2025 年 2 月发布，是一个**运行在本地的 AI Agent**。它会持续理解你的上下文，生成代码，执行验证，循环迭代直到任务完成。

### 1.2 核心优势

| 优势 | 说明 |
|------|------|
| 🔓 本地权限 | 直接读写本地文件，执行终端命令 |
| 🎯 优秀 Harness | 同模型下工具调用更精准，效果更优 |

### 1.3 适用场景

| 场景 | 典型用法 |
|------|----------|
| 💻 编程开发 | 零基础也能写出可用应用 |
| 📝 文案写作 | 资料整理、内容创作 |
| 📊 数据分析 | 表格生成、报告输出 |
| 🎨 内容创作 | 多模态内容生产 |

### 1.4 使用平台

官方桌面端、网页版、IDE 插件、终端均可使用。**终端原生版功能最全**。

## 二、安装与配置

### 2.1 环境准备

1. 下载 IDE（推荐 Cursor / Antigravity / Trae）
2. 创建项目文件夹

### 2.2 安装

| 方式 | 操作 |
|------|------|
| 一行命令安装 | 官网复制命令 → 终端执行 |
| AI 协助安装 | 让 IDE Agent 自动安装 |
| 验证安装 | `claude --version` 查看版本号 |

### 2.3 模型配置

| 类型 | 配置方式 |
|------|----------|
| 官方模型 | `cloud login` 登录即可 |
| 第三方 API | CC Switch 管理，填入 API Key + Base URL |

### 2.4 启动

```
输入 claude → 选主题 → 同意安全提示 → 信任文件夹
```

## 三、基础使用与交互

### 3.1 权限模式

`Shift + Tab` 切换，四种模式安全等级从低到高：

| 模式 | 行为 | 风险 |
|------|------|------|
| 🟢 计划模式 | 先出计划，确认后执行 | ⭐ 最安全 |
| 🟡 默认模式 | 自主判断是否询问 | ⭐⭐ |
| 🟠 自动编辑 | 改文件不询问，命令需确认 | ⭐⭐⭐ |
| 🔴 全自动 | 启动加参数，完全自主 | ⭐⭐⭐⭐ |

### 3.2 交互方式

| 方式 | 操作 |
|------|------|
| 💬 文字对话 | 直接描述需求 |
| 📎 @文件 | 精准指定文件，省 Token |
| 🖼️ 图片输入 | 拖拽或 Ctrl+V 粘贴，多模态理解 |
| ↵ 换行 | Mac：Option+Enter；Windows：Ctrl+Enter |

### 3.3 常用斜杠命令

| 命令 | 功能 |
|------|------|
| `/model` | 切换模型 |
| `/btw` | 无关问题隔离对话 |
| `/simplify` | 代码优化审核 |
| `/help` | 查看所有命令 |

### 3.4 项目运行

| 操作 | 方法 |
|------|------|
| 新终端执行 | 直接输入命令 |
| CC 内执行 | 命令前加 `!` |
| 后台运行 | `Ctrl + B` |

## 四、管控与维护

### 4.1 回滚机制

| 方式 | 操作 |
|------|------|
| ⏪ 快速回滚 | 双击 ESC 或输入 `//rewind` |
| 🗂️ 版本管理 | Git（本地 + GitHub 远程存档） |

### 4.2 上下文管理

> ⚠️ 上下文过长会导致响应变慢、效果下降

| 命令 | 功能 |
|------|------|
| `/compact` | 主动压缩上下文 |
| `/clear` | 清空上下文 |
| `/context` | 查看上下文占用 |
| 百分比显示 | 开启实时监控 |

### 4.3 对话恢复

| 方式 | 操作 |
|------|------|
| 恢复历史对话 | `/resolve` |
| 继续上次对话 | `claude -c` |

## 五、个性化定制

### 5.1 三级 CLAUDE.md 记忆

| 级别 | 作用范围 | 用途 |
|------|----------|------|
| 🌍 全局级 | 所有项目 | 通用规则 |
| 📁 项目级 | 当前项目 | `/init` 初始化，项目专属信息 |
| 📂 文件夹级 | 子目录 | 子目录规则 |

### 5.2 自动记忆（Auto Memory）

| 操作 | 说明 |
|------|------|
| 开启 | `/memory` → 打开 Auto Memory |
| 记录内容 | 用户习惯、反馈、项目进度、资源索引 |
| 管理 | 查看 / 删除记忆 |

### 5.3 自定义记忆

自建规范文件，在 CLAUDE.md 中指定读取规则。

## 六、高阶扩展

### 6.1 Skills（技能）

| 项目 | 说明 |
|------|------|
| 类型 | 知识型、流程型、工具型、混合型 |
| 安装 | 手动放指定目录 / Final Skill 自动安装 |
| 调用 | 自动触发 / 手动调用 / 指令指定 |
| 创建 | Skill Creator 生成自定义技能 |

### 6.2 外部工具连接

| 工具 | 说明 |
|------|------|
| 🔌 MCP | AI 与外部工具的转接器 |
| ⌨️ CLI | 命令行工具，高效调用外部服务（飞书、Open CLI） |

### 6.3 子 Agent（Sub Agent）

并行执行任务，独立上下文。支持自动派生或手动创建。

### 6.4 Hook（触发器）

设定条件自动执行任务——提交前检查、代码格式化等。

### 6.5 插件

`/plugin` 管理，可打包 Skills、Sub Agent、MCP 等。

## 七、学习路径

### 🟢 新手阶段

```
安装 → 基础交互 → 权限模式 → 简单项目
```

**重点**：熟悉对话方式、权限切换、基本命令。

### 🟡 进阶阶段

```
管控回滚 → 上下文管理 → 个性化记忆
```

**重点**：掌握 CLAUDE.md 配置、Auto Memory、上下文优化。

### 🔴 高阶阶段

```
Skills → 外部工具 → 子 Agent → Hook → 插件
```

**重点**：构建个人工作流，打通外部生态，实现自动化。

## 快速参考

| 场景 | 操作 |
|------|------|
| 🆕 刚开始用 | `claude` 启动，Plan Mode 最安全 |
| ⌨️ 写代码 | 描述需求 + @文件 指定上下文 |
| 🚀 跑项目 | `!` + 命令 或 Ctrl+B 后台 |
| 🐛 出错了 | 双击 ESC 回滚 / Git 版本管理 |
| 📈 上下文满 | `/compact` 压缩 或 `/clear` 清空 |
| 🔄 换模型 | `/model` 切换 |
| 📦 多文件改动 | 全自动模式 + Git 版本管理兜底 |
| ⚙️ 定制化 | 写 CLAUDE.md + 开启 Auto Memory |

---

> 💡 **Tips**：Claude Code 的核心理念是「对话即编程」。善用上下文管理和权限模式，既能高效开发，又能保障安全。
